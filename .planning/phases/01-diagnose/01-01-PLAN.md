IMPORTANT: This is a READ-ONLY diagnostic phase. Do NOT run `firebase deploy` or modify any production rules.

Step 1 — Refresh Firebase CLI token and extract it:
```bash
firebase projects:list --project level-up-app-c9f47 > /dev/null 2>&1
TOKEN=$(node -e "const { configstore } = require('firebase-tools/lib/configstore'); console.log(configstore.get('tokens').access_token);")
```

Step 2 — Fetch live releases from REST API:
```bash
RELEASES=$(curl -s -H "Authorization: Bearer $TOKEN" \
  "https://firebaserules.googleapis.com/v1/projects/level-up-app-c9f47/releases")
```

Step 3 — Extract ruleset IDs for Firestore and Storage:
```bash
FIRESTORE_RULESET=$(echo "$RELEASES" | node -e "
  const d = JSON.parse(require('fs').readFileSync('/dev/stdin','utf8'));
  const r = d.releases.find(r => r.name.includes('cloud.firestore'));
  console.log(r.rulesetName.split('/').pop());
")
STORAGE_RULESET=$(echo "$RELEASES" | node -e "
  const d = JSON.parse(require('fs').readFileSync('/dev/stdin','utf8'));
  const r = d.releases.find(r => r.name.includes('firebase.storage'));
  console.log(r.rulesetName.split('/').pop());
")
```

Step 4 — Fetch and save live rule content (use JSON.parse to handle escaping correctly):
```bash
curl -s -H "Authorization: Bearer $TOKEN" \
  "https://firebaserules.googleapis.com/v1/projects/level-up-app-c9f47/rulesets/$FIRESTORE_RULESET" \
  | node -e "const d=JSON.parse(require('fs').readFileSync('/dev/stdin','utf8')); process.stdout.write(d.source.files[0].content);" \
  > /tmp/live-firestore.rules

curl -s -H "Authorization: Bearer $TOKEN" \
  "https://firebaserules.googleapis.com/v1/projects/level-up-app-c9f47/rulesets/$STORAGE_RULESET" \
  | node -e "const d=JSON.parse(require('fs').readFileSync('/dev/stdin','utf8')); process.stdout.write(d.source.files[0].content);" \
  > /tmp/live-storage.rules
```

Step 5 — Run all diffs (save output to files for Task 2):
```bash
# Firestore: Live vs PWA
diff -u /tmp/live-firestore.rules /Users/jimrodarte/Documents/GitHub/Level-Up-App/firestore.rules > /tmp/diff-firestore-live-vs-pwa.txt 2>&1 || true

# Firestore: Live vs Native App
diff -u /tmp/live-firestore.rules "/Users/jimrodarte/Level Up App/LevelUp-Fresh/firestore.rules" > /tmp/diff-firestore-live-vs-native.txt 2>&1 || true

# Firestore: PWA vs Native App (for merge planning)
diff -u /Users/jimrodarte/Documents/GitHub/Level-Up-App/firestore.rules "/Users/jimrodarte/Level Up App/LevelUp-Fresh/firestore.rules" > /tmp/diff-firestore-pwa-vs-native.txt 2>&1 || true

# Storage: Live vs Native App (PWA has no storage.rules)
diff -u /tmp/live-storage.rules "/Users/jimrodarte/Level Up App/LevelUp-Fresh/storage.rules" > /tmp/diff-storage-live-vs-native.txt 2>&1 || true
```

Step 6 — Verify Cloud Functions are deployed:
```bash
firebase functions:list --project level-up-app-c9f47 > /tmp/functions-list.txt 2>&1
```

Step 7 — Also capture the release timestamps from the RELEASES JSON for the diagnosis report:
```bash
echo "$RELEASES" | node -e "
  const d = JSON.parse(require('fs').readFileSync('/dev/stdin','utf8'));
  d.releases.forEach(r => console.log(r.name + ' | ' + r.updateTime));
" > /tmp/release-timestamps.txt
```

Note on OAuth token expiry: If REST API returns 401, re-run Step 1 to refresh the token. The token typically expires after 1 hour.
