**Create `storage.rules`** in the repo root (new file). Per user decision and research:

```
rules_version = '2';
service firebase.storage {
  match /b/{bucket}/o {
    // User profile photos -- PUBLIC read for external embeds (Squarespace, Salesforce)
    // Cloud Functions (coaches, students, getPhoto) return direct Storage URLs
    // that browsers load without Firebase auth -- public read is required.
    match /users/{userId}/{allPaths=**} {
      allow read: if true;
      allow write: if request.auth != null
        && request.auth.uid == userId
        && request.resource.contentType.matches('image/(jpeg|png|webp)')
        && request.resource.size < 5 * 1024 * 1024;
    }

    // Event header images -- auth-required (not used by external integrations)
    match /events/{eventId}/{allPaths=**} {
      allow read: if request.auth != null;
      allow write: if request.auth != null
        && request.resource.contentType.matches('image/(jpeg|png|webp)')
        && request.resource.size < 5 * 1024 * 1024;
    }

    // Default: deny all other paths
    match /{allPaths=**} {
      allow read, write: if false;
    }
  }
}
```

Key difference from native app's storage.rules: `allow read: if true` on `users/{userId}` paths (was `if request.auth != null`). Event images stay auth-required per user decision (not used by external integrations). Write rules are unchanged from native app.

**Update `firebase.json`:** Add `"storage"` key at the top level, after the `"firestore"` key:

```json
}
```

Keep all existing keys (`firestore`, `hosting`, `functions`) unchanged. Only add the `storage` key.

- Do NOT change the write rules -- they match the native app and are correct.
