# Phase 02.1: Persistent Native App Download Promotion Banner in PWA - Context

**Gathered:** 2026-03-21
**Status:** Ready for planning

<domain>
## Phase Boundary

Make it clear and obvious to every PWA user that iOS and Android native apps are available for download. Upgrade the existing AppStoreBanner component to be persistent (per-session dismissal only), visible on all platforms including desktop, and remove the competing NotificationPrompt modal entirely.

</domain>

<decisions>
## Implementation Decisions

### Desktop visibility
- Show the download banner on ALL platforms — iOS, Android, AND desktop
- Desktop users see a QR code that links to `/download.html` (existing page auto-detects platform and redirects to correct store)
- Mobile users continue to see platform-specific button (iOS → App Store, Android → Google Play) — current behavior

### Banner prominence
- Full-width slim bar at the top of the page — same style as current implementation
- Fixed to top, pushes content down
- Brand colors: Primary Blue (#18264E) background, white text, Coral (#F15F5E) CTA button
- Appears on ALL authenticated pages, no exceptions

### Persistence & dismissal
- Banner is dismissible via close button, but only hides for the current session
- Use sessionStorage (not localStorage) — banner returns every time the user closes the tab/browser and comes back
- Remove the 14-day localStorage cooldown entirely
- Always show to all PWA users — no attempt to detect if native app is installed
- Goal is maximum persistent visibility to drive migration to native apps

### Notification prompt removal
- Remove `NotificationPrompt.jsx` component entirely (not just disable)
- Remove all references to it from App.jsx and anywhere else it's imported
- Native app handles push notifications now — no need to prompt in PWA

### Claude's Discretion
- Messaging approach (static vs rotating value props)
- Session boundary definition (sessionStorage vs equivalent)
- QR code generation approach for desktop
- Exact spacing and typography within the banner
- Animation/transition behavior

</decisions>

<specifics>
## Specific Ideas

- QR code should link to `/download.html` which already handles platform detection and auto-redirects to the correct app store
- The shift from localStorage (14-day) to sessionStorage (per-session) is the key behavioral change making this "persistent"
- This is a migration push — the tone should encourage moving to the native app, not just inform about it

</specifics>
